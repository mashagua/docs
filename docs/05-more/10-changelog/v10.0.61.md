# DesireCore v10.0.61

**发布日期**: 2026-05-19

![v10.0.61 更新概览](/img/changelog/v10.0.61.png)

## 新功能
- **团队 Git 版本管理**：现在你可以把团队作为可版本化资源进行管理，直接连接 GitHub、Gitee 或 Gitea 远程仓库，完成推送、拉取、发布和一键 Fork 安装团队。
- **高精度数学计算工具**：Agent 现在会使用新的高精度数学计算能力处理数值运算、矩阵、单位换算和分数等场景，减少复杂计算中的结果误差。
- **模型能力指示器**：聊天中的模型选择器会显示模型是否支持视觉等能力，让你在切换模型前就能知道它能做什么、不能做什么。
- **聊天文件粘贴引用**：你现在可以直接把本地文件或文件夹粘贴到聊天输入框中作为文件引用发送，同时图片粘贴仍保持原有上传方式。
- **沉浸式语音交互**：沉浸式模式新增语音输入与语音播报能力，支持持续语音识别、语音合成和更自然的对话反馈。
- **AI 下一条消息预测**：当助手回复结束后，输入框上方会出现智能预测建议，你可以一键采纳常见的下一步输入。
- **沉浸式决策浮层**：在沉浸式模式下，审批卡片和提问卡片会以浮层形式显示，不会再因为进入沉浸式而看不到需要处理的关键交互。
- **/steer 中途干预命令**：你可以在 Agent 执行过程中使用 `/steer` 发送新的干预指令，并在聊天记录中看到带有“已干预”标记的消息。
- **AskUserQuestion 超时机制**：Agent 向你发起问题时现在可以设置超时倒计时，超时后任务不会一直卡住等待。
- **技能目录上下文注入**：技能现在可以自动感知自身目录和资源清单，使用依赖脚本、参考文件和附属资源时更稳定。
- **工作流校验增强**：工作流在运行前会更早发现条件分支无后续、节点字段误填等问题，减少“看似成功但实际没跑完”的情况。
- **算力 Token Plan 支持**：算力配置现已支持 Token Plan 相关字段，为新的团队计量方案提供兼容支持。

## 修复
- 修复了部分模型回复初始为空、随后流式补齐时，消息组件可能报错或崩溃的问题。
- 修复了通知中心里的 Markdown 正文不能正确显示的问题，现在加粗、行内代码和列表等格式可正常预览。
- 修复了 SuperDoc 中行内 diff 装饰有时不显示的问题，让文档编辑后的增删改标记更稳定出现。
- 修复了编辑 `.md` 文件后有时会出现空白助手气泡的问题，并在模型未主动发送附件时自动补发可打开的文件卡片。
- 修复了运行状态面板里的技能名称和描述不会随界面语言切换的问题。
- 修复了快速切换语言或智能体时，技能列表可能被旧请求结果覆盖的问题。
- 修复了合并消息视图中重复编辑同一条用户消息后，旧版本消息仍残留的问题。
- 修复了历史截断后旧话题记忆仍可能继续注入后续对话的问题。
- 修复了设置页点击版本号检查更新后，界面可能一直停留在“检查中”的问题。
- 修复了手动版本切换弹窗宽度不足、加载状态不明显以及下载提示混乱的问题。
- 修复了算力配置与节点接口在特定启动路径下返回 500 的问题。
- 修复了默认仓库 zip 在打包后的路径解析错误，避免首次启动时内置仓库无法正确解压。
- 修复了 TLS 证书过期后续期依赖其他服务启动顺序的问题，避免本地服务证书失效导致连接异常。
- 修复了 Windows 自动更新中多种导致升级失败的问题，显著提升了 Program Files 和普通安装目录下的升级成功率。
- 修复了市场仓库缓存脏状态导致同步失败的问题，并增强了异常缓存的自动自愈能力。
- 修复了 zip 解压后 Unix 可执行权限丢失，进而引发仓库被误判为本地已修改的问题。
- 修复了依赖升级后主进程构建失败、应用无法启动的问题。
- 修复了运行任务在应用重启后仍错误显示为“执行中”的问题，并补齐了 webhook 同类状态异常。
- 修复了定时调度任务可能长期卡在旧 run 上、不再生成新执行记录的问题。
- 修复了兼容代理默认注入服务端网页搜索工具后，某些模型回复被搜索结果文本污染的问题。
- 修复了流式响应被网络中断时仍提示你检查模型配置的误导性报错，现在能更准确区分网络中断与模型空响应。
- 修复了 AskUserQuestion 在跳过、超时、断线重连和提交失败等场景下的状态显示与回滚问题，避免问答卡片卡死或状态错乱。
- 修复了沉浸式模式下提交或跳过问题卡片时的若干竞态问题，减少误回滚和无响应情况。

## 改进
- **AskUserQuestion 交互体验**：问答卡片现在会实时提示还剩多少题未答、支持快捷键提交，并会自动定位到未完成的问题，让长表单更容易填完。
- 视觉能力受限的模型现在会自动禁用图片上传、截图、图片粘贴和拖拽等入口，避免你选择了不支持视觉的模型后才发现无法使用。
- 沉浸式模式增加了更多进入方式，包括快捷键、从附件直接进入，以及 Agent 主动请求切换，使用路径更顺手。
- 市场与内置仓库的启动同步流程更稳定，离线首次安装时也能更可靠地拿到完整默认内容。
- 审计日志现在默认保留完整历史，并重新记录流式返回分片，方便后续追踪与排查。
- 聊天中 `/steer` 干预消息现在会持久化到历史记录中，刷新或重启后也能追溯你中途改过哪些指令。
- 技能自动同步逻辑优化后，启动更高效，后台定时同步不再反复做无意义的内置内容扫描。
- 模型与依赖升级后，整体运行环境、图标资源、搜索能力和桌面壳层兼容性得到提升。
