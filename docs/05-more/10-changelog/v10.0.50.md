# DesireCore v10.0.50

**发布日期**: 2026-05-09

![v10.0.50 更新概览](/img/changelog/v10.0.50.png)

## 新功能
- **对话压缩**：现在可以通过 `/compact` 命令或聊天页菜单手动压缩长对话，在保留最近上下文的同时把较早内容整理为摘要，继续对话更连贯。
- **智能体冲突解决器**：当智能体同步更新发生文件冲突时，你现在可以直接在应用内查看本地与远端差异，并逐个选择保留本地、保留远端或手动编辑解决。
- **技能多语言注入**：应用会根据你当前的客户端语言，向智能体注入对应语言的 Skill 内容，让中文界面下的技能提示也能保持中文。
- **YAML Frontmatter 渲染**：在编辑 SKILL.md 等文件时，YAML frontmatter 现在会正确显示为结构化内容，不再被误渲染成分隔线。

## 修复
- 手动执行 `/compact` 时，如果当前对话内容太少而无需压缩，提示消息现在会自动消失，不再一直转圈像界面卡住。
- 修复了智能体“检测更新”在网络异常时误报“已是最新版本”的问题，并且手动检查会直接获取最新远端信息而不是读旧缓存。
- 修复了智能体自动更新冲突信息丢失的问题，现在会明确显示冲突文件，并持续显示冲突状态直到你处理完成。
- 修复了更新下载安装偶发卡在 0% 不动的问题，下载停滞后会自动切换到备用源继续尝试。
- 修复了后台心跳任务的重试提示混入当前聊天窗口的问题，避免对话里出现无法清除的重试消息。
- 修复了 Agent 写入记忆文件时误弹出 Markdown 编辑器的问题，记忆内容会按预期以内存卡片形式呈现。
- 修复了某些工具调用配对异常引发的模型报错，减少长对话中出现图片数据格式错误的情况。
- 修复了新建智能体后 Git 仓库初始化的竞态问题，创建后立即执行 Git 相关操作会更稳定。
- 修复了桌面应用长期运行时部分会话无法及时释放的问题，降低长时间使用后的资源占用风险。
- 修复了心跳通知写文件时 `~` 路径未正确展开的问题，相关文件现在会保存到正确的绝对路径。
- 修复了社会化关系同步不及时的问题，新安装智能体、用户 `@` 提及和关系时间线现在能更快反映最新变化。

## 改进
- **更新体验**：在智能体“检测更新”过程中，按钮现在会显示明确的加载状态并防止重复点击。
- **冲突状态提示**：聊天头部的智能体名称旁重新加入了冲突徽章，让你更容易发现同步冲突并进入处理流程。
- **上下文管理**：系统会更早开始自动压缩上下文，减少超长对话造成的上下文膨胀和响应质量下降。
- **长文本输出控制**：智能体单次任务现在默认有更合理的轮次上限，并会自动清理异常重复的回复内容。
- **心跳通知卡片**：较长的心跳通知内容现在会自动折叠，需要时可展开查看，阅读更清爽。
- **心跳历史可追溯**：心跳执行结果会生成并维护历史汇总，方便后续回顾与引用。
- **代码托管文案**：代码托管面板中的“添加仓库”现已统一为“添加托管源”，概念表达更清晰。
- **记忆写入约束**：系统进一步限制记忆只能写入 AgentFS 指定位置，减少误写到工作目录的情况。
- **更新发现速度**：客户端会按当前操作系统获取可用版本列表，通常能更早看到适用于本机的新版本。
- **开发版与正式版隔离**：开发版与正式版现在可以在同一台设备上同时运行，彼此的数据和运行状态互不干扰。
