# DesireCore v10.0.46

**发布日期**: 2026-05-05

![v10.0.46 更新概览](/img/changelog/v10.0.46.png)

## 新功能
- **界面多语言支持**：前端主要界面文案已全面支持中英文切换，切换语言后包括窗口标题栏菜单在内的界面内容都会同步更新。
- **Skill 单独导出**：现在可以直接从 Skill 详情页单独导出全局 Skill 或智能体内嵌 Skill，并保留脚本、参考资料等完整目录结构。
- **市场内容多语言展示**：智能体市场与技能市场现已支持按当前语言自动显示对应内容，中英文搜索也都能命中相关条目。
- **新增音乐生成技能**：新增 MiniMax 音乐生成技能，可直接通过对话生成音乐内容。
- **新增运行环境相关技能**：新增 Python、Node.js 与开发环境准备相关技能，帮助智能体更完整地处理开发与运行环境任务。

## 修复
- 从市场安装或新建智能体后，现在会立即正确显示在线状态，不再出现“刚导入显示离线、刷新后才恢复”的情况。
- 修复了大型智能体导入过程中列表状态偶发闪烁或错误显示的问题，导入过程中的状态更新更稳定。
- 修复了媒体生成请求被误判为文档创作的问题，生成的图片、音频、视频现在会优先在聊天中内联展示和播放。
- 修复了团队导入后共享工作目录配置丢失的问题，导入团队后相关工作目录会一并恢复。
- 修复了导出单个智能体时目录标识丢失的问题，重新导入后能保留原有智能体目录 ID。
- 修复了应用商店在遇到未知安装状态或分类数据时可能直接崩溃的问题，现在最多隐藏异常徽标而不会影响页面使用。
- 修复了智能体管理列表中卡片内容未顶部对齐的问题，同一行卡片的头像和状态信息显示更整齐。
- 修复了智能体配置表单中运行时环境注入选项默认值未正确显示的问题，新建或编辑时默认勾选状态会正确呈现。
- 修复了部分低频场景下由路径解析错误导致的功能异常，提升了导出、关系广播和仓库配置等场景的稳定性。

## 改进
- 智能体会根据自己实际可用的工具动态理解自身能力，减少无效提示信息，让受限工具配置下的对话更准确、更省上下文。
- 智能体资料页展示的可用工具列表现在与后端实际生效的工具权限保持一致，也能正确显示 MCP 工具。
- 对话记忆压缩策略现在会真正按照智能体配置生效，支持关闭自动压缩或自定义触发阈值。
- 内置工具命名已统一，历史旧名称仍可兼容使用，相关技能与对话展示也更加一致。
- 全局技能库已同步更新为最新版本，包含更多双语技能说明、更新后的工具调用方式以及更完整的工作流技能内容。
