# 术语表

本术语表收录了 DesireCore 中的核心概念和专有名词，按英文字母顺序排列。

## A

### AgentFS（智能体文件系统）
DesireCore 的文件系统层，每个智能体都是一个 AgentFS 仓库。包含智能体的身份、人格、规则、技能、记忆等所有数据，以文件形式存储并通过 Git 进行版本管理。

### Agent（智能体）
DesireCore 中的核心实体，是一个可培养、可托管、可治理的数字同伴。不是简单的聊天机器人，而是能接受委派、执行任务、交付回执的拟人化助手。

### ASR（语音识别）
Automatic Speech Recognition 的缩写，将语音转换为文字的技术。DesireCore 支持通过多个供应商的 ASR 模型实现语音输入。

### Assign（下达任务）
六原语之外的核心交互动作，用户向智能体委派具体任务。与传统的"提问"不同，"下达任务"意味着智能体需要完整地规划、执行并交付结果。

## B

### BYOK（Bring Your Own Key）
"自带钥匙"模式。用户使用自己的 AI 供应商 API Key，DesireCore 不做中间代理，费用直接由供应商向用户结算。

## C

### Clarify（问）
六原语之一。智能体在信息不完整时主动向用户追问，补齐任务所需的关键条件。好的智能体会在行动前先确认，而不是靠猜。

### Companion（同伴）
数字同伴（Digital Companion）的简称。DesireCore 对智能体的另一种称呼，强调其"长期陪伴、持续成长"的特质。

### Compute（算力）
AI 模型运行所需的计算资源。在 DesireCore 中，算力通过 BYOK 模式由用户自行配置，支持多家供应商和多种模型。

### Confirm（确认）
用户对智能体操作的审批动作。在人闸门步骤中，智能体暂停执行并等待用户确认后才继续。

### Controllable（可控）
三层可控性的第二层。用户可以随时介入、确认或阻断智能体的行为，通过权限分级（allow/ask/deny）精确控制自主程度。

### Core Memory（核心记忆）
智能体的核心身份信息和长期知识，存储在 AgentFS 的 `memory/` 目录中。包括角色定义、专业知识等不常变化的信息。

## D

### Delegation（委派）
DesireCore 的核心交互范式。不是"我问你答"，而是"我教你做"。用户将任务委派给智能体，智能体自主规划和执行，最终交付结果和回执。

### Demonstrate（演/给示例）
六原语之一。用户通过提供示例、反例或历史材料，让智能体通过模仿和归纳来学习。

### Diff
文件或内容的变更记录，用红色标注删除内容、绿色标注新增内容。DesireCore 中智能体的每次修改都以 Diff 形式可视化展示。

## E

### Embedding（嵌入/向量化）
将文本转换为数值向量的过程，用于语义搜索和相似度匹配。DesireCore 支持多家供应商的嵌入模型。

### Execute（做）
六原语之一。智能体在获得计划确认后，在允许的权限和边界内行动、调用工具、推进任务。

## F

### Feedback（反馈）
用户对智能体执行结果的评价和纠正。正面反馈强化行为，负面反馈触发调整，是智能体进化的重要信号。

### Flexible Step（灵活步骤）
三种步骤类型之一。需要 AI 模型推理、生成或归纳的步骤，智能体在行为准则框架内有自主判断空间。

### Forget（遗忘）
主动删除智能体的特定记忆或学习成果。用户可以让智能体"忘记"某些信息，实现记忆的可管理性。

## H

### Hardened Step（固化步骤）
三种步骤类型之一。基于规则、决策树、固定模板的确定性步骤，结果完全可预测、可重复，不涉及 AI 推理。

### Heartbeat（心跳）
智能体的主动监控机制。智能体可以定期检查特定数据源或条件，在满足触发规则时主动通知用户或执行操作。

### Human Gate（人闸门）
三种步骤类型之一。需要用户亲自确认的步骤。执行到此步骤时自动暂停，等待用户明确指示后才继续。

## M

### MCP（Model Context Protocol）
模型上下文协议。由 Anthropic 提出的开放标准，允许 AI 模型连接外部工具和数据源。DesireCore 支持 MCP 协议来扩展智能体的工具能力。

## P

### Persona（人格档案）
智能体的性格定义文件（`persona.md`）。包含沟通风格、决策偏好、输出规范等，是智能体"像谁"的核心定义。不是"人设文案"，而是可执行的行为规范。

### Plan（立）
六原语之一。智能体给出执行计划，拆解步骤、标注风险点和需要用户确认的环节。

### Principles（行为准则）
智能体的行为规则文件（`principles.md`）。定义了"必须做什么""绝不做什么""什么时候找人确认"等规则。

## R

### Receipt（回执）
智能体完成任务后生成的详细记录，包含输入输出摘要、工具调用清单、检索轨迹、步骤类型统计等信息。是"可信委派"的证据链。

### Reflect（复）
六原语之一。智能体提交回执、复盘偏差、把新经验写回行为手册，并提出改进建议。

### Relational Memory（关系记忆）
记录用户与智能体之间的互动历史和个性化信息。存储在 `users/<user_id>/agents/<agent_id>/memory/` 中。

### Reversible（可逆）
三层可控性的第三层。任何操作都可以撤销或回滚，包括逐步回滚、按轮回滚和整会话回滚。

## S

### Self-Evolution（自我进化）
智能体在交互中持续学习和成长的能力。包括隐式学习、显式教学、复盘进化和协同进化四种模式。

### Shared Memory（共享记忆）
团队共享的知识和经验，多个用户和智能体可以共同访问。用于沉淀团队规范、最佳实践等。

### Six Primitives（六原语）
DesireCore 定义的六个基本交互原语：教（Teach）、演（Demonstrate）、问（Clarify）、立（Plan）、做（Execute）、复（Reflect）。

### Skill（技能）
智能体学会的特定能力模块。以文件夹形式存储在 AgentFS 的 `skills/` 目录中，包含指令文件（SKILL.md）和可选的脚本、参考资料等。

### Super Document（超级文书）
DesireCore 的协作写作功能。基于 Markdown 编辑器，支持 AI 辅助编写、Diff 可视化、逐条审阅、版本回溯，像 Code Review 一样审文档。

## T

### Teach（教）
六原语之一。用户告诉智能体怎么做、为什么这么做、什么情况例外。教学内容会被持久化为行为规范。

### Team Member Lock（团队成员锁定）
团队对每位成员智能体的精确版本指向。即使成员智能体在自己的仓库里继续迭代,团队默认仍指向锁定的版本,保证别人下次安装时能装到一模一样的组合。需要同步到成员最新版本时,使用"升级成员"操作。

### Team Release（团队发布）
为团队打一个完整的版本号(例如 v0.1.1)、生成变更日志、并同步到远程。发布后这个版本可以被分享、回退、或作为参考点。

### Team Version Management（团队版本管理）
把整支智能体团队当作一个整体进行版本管理的能力,包括组织变化(加成员/换队长/解散)的自动记录、推送到 GitHub/Gitee 远程仓库、发布团队新版本、从他人链接一键 Fork 安装等。与单个智能体的版本管理是两层独立体系。

### Tool（工具）
智能体可以调用的外部能力。包括内置工具（文件读写、搜索等）、MCP 工具和自定义工具。通过 Tool Registry 统一注册和管理。

### Tool Registry（工具注册中心）
统一管理所有工具的注册、发现和调用的系统。将内置工具、MCP 工具和技能定义的工具统一到一个层面管理。

### TTS（语音合成）
Text-to-Speech 的缩写，将文字转换为语音的技术。

## V

### Visible（可见）
三层可控性的第一层。智能体的所有行为对用户完全透明，包括实时状态、决策过程、修改内容和工具调用。

## W

### Workflow（工作流）
将多个步骤按逻辑组织起来的可复用执行单元。支持用自然语言描述意图，智能体自主编排（Vibe Workflow）。
