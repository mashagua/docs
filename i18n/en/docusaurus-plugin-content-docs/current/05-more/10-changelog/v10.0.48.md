# DesireCore v10.0.48

**Release Date**: 2026-05-08

![v10.0.48 更新概览](/img/changelog/v10.0.48-en.png)

## New Features
- **Heartbeat notification file links**: Agents can now attach generated files to heartbeat notifications. You can click to open them directly from the notification card or the history. Markdown files can be viewed with SuperDoc, while other files will open using the system default application.
- **Concurrent heartbeat support for multiple agents**: Scheduled heartbeat tasks for multiple agents can now run more reliably at the same time without competing with or interfering with one another.

## Fixes
- Workflow now blocks graph structure issues that are not supported at runtime earlier during creation or validation, preventing cases where “validation passes but execution behaves abnormally.”
- Workflow branch preview and validation messages are now clearer, making it easier to see conditional branches, fallback paths, and referenced variables, reducing the cost of troubleshooting configuration issues.

## Improvements
- Heartbeat notification logic now better reflects actual results: it sends reminders more proactively when there are results, and stays silent only after confirming that nothing has changed, reducing unnecessary interruptions.
- Heartbeat records now retain notification summaries and related file information, making it easier to review what each heartbeat produced afterward.
- The heartbeat system has been comprehensively optimized for tool invocation, timeout control, and text content cleanup, making scheduled task execution more stable and notification content clearer.
- Workflow DSL conventions and guidance have been further standardized, making requirements for field syntax, branch constraints, and manual approval parameters clearer when building workflows.
